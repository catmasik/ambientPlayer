НАЗНАЧЕНИЕ ПРОЕКТА:  Создать  программируемый проигрыватель  нескольких файлов одновременно  по указанному  алгоритму.
Наглядный пример,  создавать звуки аналогичные этим:  https://www.youtube.com/watch?v=IvJQTWGP5Fg&t=1407s
Некоторые звуки просто зациклены, некоторые звуки вопспроизводятся с паузами,  некоторые звуки воспроизводятся с рандомными паузами.

ПОЯСНЕНИЯ ПО РЕАЛИЗАЦИИ:
На данном этапе управление плеерами осуществляется из конфигурационного  файла  program_player.cfg  в котором построчно размещены параметры для отдельных плееров, параметры разделены точкой с запятой.
 Порядок полей в файле :
    -абсолютный путь к файлу со звуком
    -повторяемость звука  !!!ПОКА НЕ ИСПОЛЬЗУЕТСЯ!!!
    -первая задержка  
    -зарежка между звуками  !!!ПОКА НЕ ИСПОЛЬЗУЕТСЯ!!!
    -громкость 
В файле есть игнор строки , что начинаются с решетки.

Что касается кода, сейчас реализовано два класса playersDirector и progamPlayer.
        /*====================================*/
        КЛАСС   progamPlayer - это враппер над обычным кьютовым медиаплеером,  в нем инкапсулируются настройки ( величина пауз и громкость ) для воспроизведения и он управляет кьют-плеером, и именно этот класс используется для переноса в треды через метод moveToThread.  
        есть слоты   play, stop,  pause  - которые предназначены для управления из-вне плеером который находится в треде 

        Для этого класса задекларированы следующие  два свойства :
            cur_task  - структура в которой находятся параметры для управления плеером . Поля структуры :
                        audio_file_path;  // путь к аудиофайлу
                        repeatability;  // кол-во воспроизведений  ( 0 - бесконечно, 1- 1 раз и тд.  )  !!!ПОКА НЕ ИСПОЛЬЗУЕТСЯ!!!
                        first_delay;    // задержка до первого воспроизведения  ( 100   -  секунды в десятых или  rnd(200-3000)  ) 
                        delay       //  задержка между воспроизведениями  ( 100   -  секунды в дес¤тых или  rnd(200-3000)  )   !!!ПОКА НЕ ИСПОЛЬЗУЕТСЯ!!!
                        volume  // громкость плеера   (  0-100 )
           *player  //  указатель  на иницированный  кьют-плеер для которого и создан данный  враппер 

        /*====================================*/
        КЛАСС  playersDirector -  это класс в  который планируется спрятать всю логику  для управления  плеерами и оставить для паблик доступа только минимум слотов или методов
            для этого  класса реализованы след.  методы:
                        loadPlayersArray ( QString file_name )  - метод для чтения конфигурационного файла и перенос параметров запуска плееров в вектор, который потом  будет разобран на потоки 
                        createPleyers  -  метод  для создания потоков плееров  из конфигурационного вектора
                        play, pause,  stop   - эти методы нужны  только для  выпуска  сигналов для плееров : playAllPlayers, pauseAllPlayers, stopAllPlayers
